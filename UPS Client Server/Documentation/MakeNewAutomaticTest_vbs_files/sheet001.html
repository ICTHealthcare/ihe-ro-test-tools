Make new automatic test
Description:
With this test scenario a new automatic test can be build.
In the dialog fill in a name for this test and after that run all scenario's that you want to run in the automatic test in the correct order.
The test that is currently loaded is saved in this xml file “<Instalation dir>\Sessionfiles\Scripts\CurrentAutomaticTestingConfiguration.xml”
If you want to stop recording scenario's to your automatic test empty the CurrentAutomaticTestName node in the xml file mentioned above.
The configuration of this automatic test will be saved in an xml file located in “<Instalation dir>\Sessionfiles\Scripts” with the name that is specified for this test.
All used messages will be stored in the "<Instalation dir>\Datasets\CLIENTSERVER\UPSMessages\ given name" directory.
In the configuration file there will be references to these messages.
With the RunAutomaticTest.vbs scenario the automatic test generated by this scenario can be executed. 
If there are more automatic test available, put the xml file name of the test that must be run in the  “<Instalation dir>\Sessionfiles\Scripts\CurrentAutomaticTestingConfiguration.xml” file.
