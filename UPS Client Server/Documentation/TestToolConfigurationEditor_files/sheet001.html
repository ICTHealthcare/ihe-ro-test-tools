Test Tool Configuration Editor
1. Configuring AE-title and Ports
1.1 Configuring the test tool as CLIENT
When the editor tool is started, the "TT as CLIENT" tab is shown (see screenshot below). The AE titles, ports and IP Addresses of the various scenario actors (and dimse commands) can be configured.
When a wrong format is typed in the fields the characters turn red and saving is not possible.
1.2 Configuring the test tool as SERVER
The test tool is either simulating a UPS server or a UPS client. In the 2nd tab the configuration of the test tool as Server is shown. In some scenarios, when acting as a server it can also be acting as an OST. The test tool will use these settings to listen on a specific port. The OST is simulated as a different entity and should have a different AE title and portnumber; otherwise it is not possible to save the settings.
1.3 Configuring the SUT AE-titles and Ports
When the test tool is acting as a client it must know the parameters of the server and OST. During the scenarios the client will send C-STORE requests and C-MOVE requests to the OST. And it will also send a C-FIND request, C-MOVE request and N-SET and N-ACTION requests to the server. For both the OST as the server it could very well be that only one AE title and portnumber is needed, therefore it is possible to use the same settings for different dimse commands.
When the test tool is acting as a server, it will most of the times react to incoming request. One exception is when it will receive a C-MOVE request from the client, it should know to which port number it should send the C-STORE requests.

2. Adding or removing transfer syntaxes
It is possible to select transfer syntaxes for both sender and receiver.
Default transfer syntaxes for the sender:
1.2.840.10008.1.2.1 (Explicit VR Little Endian (ELE))
1.2.840.10008.1.2 (Implicit VR Little Endian: Default Transfer Syntax for DICOM (ILE))
1.2.840.10008.1.2.2 (Explicit VR Big Endian (EBE))
Default transfer syntaxes for the receiver:
1.2.840.10008.1.2.1 (Explicit VR Little Endian (ELE))
1.2.840.10008.1.2 (Implicit VR Little Endian: Default Transfer Syntax for DICOM (ILE))
1.2.840.10008.1.2.2 (Explicit VR Big Endian (EBE))
3. Debug settings
In this tab it is possible to enable/disable some debug settings:
-       Report Debug messages: check to see debug messages in the DVT logging, uncheck to disable them
-       Report TBD Rule: enable this to let all TBD rules report an error message when executed, uncheck to disable these messages.
4. Set dataset
It is possible to use your own datasets for testing. With the first button the objects on the OST can be set.
The 2nd button is for the objects on the UPS Server, the last required button is to set your own UPS'es.
The last button is the clear button, all given objects will be deleted.
5. AE Title's
A UPS Client can subscribe to an UPS (Send-10-UPS-SubscribeOrUnsubscribe.vbs scenario).
If this is done an AE-title must be given by the UPS client.
In the AE-titles's tab of the configuration editor all AE-titles are configured with an Ip adres and a port number.
An entry can be added by filling in the three textboxes and hit the 'Add AE-title button'.
The UPS server test tool will use this list to update subscribers if an UPS is updated that has subscribers.
