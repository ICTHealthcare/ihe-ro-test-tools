Send UPS Worklist Query
Instructions for use test scenarios
Description:
This test scenario sends a C-Find to the UPS server.
1. Make sure the system under test (SUT) is active and listening.
2. Start the scenario.
3. A pop-up will appear in which you can make a UPS query for the UPS server.
4. Send the query to the UPS server by pressing ‘send’. The test tool starts to listen for incoming messages.
5. Send the C-Find responses with the UPS server.
6. The test tool automatically detects the responses and will stop.
Expected transactions from actor:
·        UPS-02 Query worklist items.
Used scenario dataset:
This scenario uses a predefined request which can be modified by the user (see below)
Additional technical information
The standard request consists of a query that uses the date/time range of the present day. It is possible to change the date/time range in the pop-up.
 The station info should either be completed filled or completely empty. The patient info can be used to further narrow the query.
 The remainder of the query has all the IHE required attributes listed in the request. The UPS server should be able to generate a value for all of them.
