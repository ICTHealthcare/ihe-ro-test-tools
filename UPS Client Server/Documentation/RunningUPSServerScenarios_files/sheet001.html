Instruction for use UPS server test scenarios
This document describes how to run the UPS server test scenarios using the DVTk based IHE-RO TestTool.
Start the IHE-RO test tool by clicking the "TestTool - UPS SERVER” shortcut.
There are two DVTk project files, one for the UPS client scenario's and one for the UPS server scenario's.
Configuration
First the Test Tool needs to be configured, starting with the connection details. This has changed because in some scenarios the test tool as to be 2 different actors.
 A Test Tool Configuration Editor is delivered along with the scenarios.
Click here to read how to use the Test Tool Configuration Editor. Note that this configuration is used instead of the session file!
Running a scenario
The test scenario will start running by double clicking on the test scenario name in the left pane (e.g. double click on the text “Receive-01-ReceiveUPSTransactions” in the left pane)
Example: Receive UPS Transactions
Start the scenario by double clicking on the text “Receive-01-ReceiveUPSTransactions” in the left pane. The following pop-ups will be displayed.
  
The testtool will wait for incoming transactions form the actor under test. While the test tool is listening a dialog will be shown as can be seen above. In the logging window the incoming and outgoing DICOM messages are shown. 
The dialog can be closed by clicking the ‘Stop listening and start validation’ button. 
The results will be shown in the test tools main window.
Test Results
The result files of all tested scenarios are located at “<Instalation dir>\TestTools UPS Client Server\ResultFiles”.
