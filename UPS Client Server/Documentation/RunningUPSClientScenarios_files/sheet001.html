Instruction for use UPS client test scenarios
This document describes how to run the send test scenarios using the DVTk based IHE-RO TestTool.
Start the IHE-RO test tools by clicking the "TestTool - UPS CLIENT” shortcut.
There are two DVTk project files, one for the UPS server scenario's and one for the UPS client scenario's.
Configuration
First the Test Tool needs to be configured, starting with the connection details. In the send scenario's the test tool is playing the UPS client actor.
 A Test Tool Configuration Editor is delivered along with the scenarios.
Click here to read how to use the Test Tool Configuration Editor. Note that this configuration is used instead of the session file!
Running a scenario
The test scenario will start running by double clicking on the test scenario name in the left pane (e.g. double click on the text “Send-02-UPS-WorklistQuery.vbs” in the left pane)
For each send scenario be sure that the system under test (SUT) is active and listening on the correct port and IP adress that is configured with the test tool configuration editor.
Example: Send UPS Worklist Query
Start the scenario by double clicking on the text “Send-02-UPS-WorklistQuery.vbs” in the left pane. The following pop-ups will be displayed.
In this scenario the test tool will send a C-Find-Request to the actor under test. In the logging window the outgoing and incoming DICOM messages are shown. 
This scenario will stop automatic after it has received the responses on the sended query.
The results will be shown in the test tools main window.
Test Results
The result files of all tested scenarios are located at “<Instalation dir>\TestTools UPS Client Server\ResultFiles”.
