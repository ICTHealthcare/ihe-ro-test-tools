Send UPS Get Information
Instructions for use test scenarios
Description:
This test scenario sends a N-Get to the UPS server.
1. Make sure the system under test (SUT) is active and listening.
2. Start the scenario.
3. A pop-up will appear in which you can add the attributes you want to receive.
4. Send the transaction to the UPS server by pressing ‘send’. The test tool starts to listen for incoming messages.
5. Send the N-Get response with the UPS server.
6. The test tool automatically detects the response and will stop executing.
Expected transactions from actor:
·          UPS-04: Get UPS Workitem Contents
Used scenario dataset:
No dataset is used in this scenario, the given attributes will be stored in an arraylist in de dicom message.
