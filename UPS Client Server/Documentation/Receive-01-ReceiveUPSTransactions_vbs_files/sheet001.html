Receive UPS transactions
Instructions for use test scenarios
Description:
This scenario hadles all UPS Server transactions. Be sure the reference UPS dataset is set,
use the testtool configuration editor.
1. Start the scenario. The test tool starts listening for incoming messages.
2. Send a UPS transaction.
3. The testtool will send a correct respons, depending on the request that is received.
4. When the communication is finished, ‘stop listening and start validation’.
5. The testtool has updated the UPS'es in the dataset cache directory.
6. Use the Show-SERVER-UPS-Status scenario to see an overview of all UPS'es on the UPS Server.
Supported transactions from actor:
UPS-01-Create UPS Workitem
UPS-02-Query UPS Workitems
UPS-03-Claim UPS Workitem
UPS-04-Get UPS Workitem Contents
UPS-05-Update UPS Workitem
UPS-06-Final Update UPS Workitem
UPS-07-Complete/Cancel UPS Workitem
UPS-08-Subscribe to UPS (Subscription)
UPS-09-UnSubscribe to UPS (Subscription)
UPS-10-Send UPS Notification
UPS-11-Cancelation request
