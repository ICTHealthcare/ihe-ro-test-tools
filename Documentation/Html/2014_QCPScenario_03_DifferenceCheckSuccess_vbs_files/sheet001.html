Test Scenario QCP Difference Check Success
Instructions for use test scenarios
Rule Overview
Description:
This test scenario tests the Quality Check Performer execution of the Difference quality check after the RT plan was retrieved.
This scenario tests the QCP UPS progress updates and retrieval of the structured report after completion of the check.
The Test tool QCR emulator and QCP configurations are configured with the testtool configuration editor
Test scenario entry point: First execute the QCP Input Objects Retrieval – test scenario.
Stage: execute
   1.   Start the test scenario, a pop-up window will appear, choose if it is a check with issues or none.
   2.   The test tool starts to listen for incoming messages. (N-EVENT_REPORT)
   3.   The QCP is expected to send a N-Event-Report request with the procedure state IN PROGRESS (RO-Q5)
   4.   The test tool will automatically respond with a N-Event-Report response
   5.   The QCP will have to send the N-Event-Report request with the procedure state COMPLETED when the check is completed.
   6.   The test tool will automatically respond with a N-Event-Report response
   7.   After the COMPLETED state is received, a N-Get request will be send  to retreive the output information sequence for where the structured report is stored. 
   8.   The QCP is expected to send the N-Get response with the requested information.
   9.   The test tool now will issue a C-Move request. (based on the previous received Output Information Sequence of the N-Get respons of RO-Q6)
   10.   The test tool starts listening for and automatically detects the C-Move response(s) and the C-STORE request.
   11.   The QCP is expected to send the C-Move response(s) and a C-Store request.
   12.   After that the structured report is received the test tool will send a N-Action unsubscribe request to the QCP.
   13.   The QCP is expected to send a N-Action response that the unsubscribe is acknowledged.
   14.   When the communication is finished, press ‘stop listening and start validation’.
   15.   The test tool will validate all transactions and the Structered Report for DICOM & IHE-compliance.
    (The validation overview can be found in the Rule Overview tab)
Structured Report validation explanation and tempate 
Notes:
        C-Move and C-Store dimse services are executed on two different associations.
        Assumption: QCP sends C-Move Responses for progress update during C-Store procedure. (some implementations of a PACS do not send 
        C-Move Responses until completion)

Additional technical information
The C-MOVE request that the test tool sends to the QCP uses the references that were found in the N-Get response of RO-Q6  the output 
information sequence should have a reference to the structured report IOD that is stored on the QCP. The “Retrieve AE Title” (0008,0054) should have 
a value that is the same as configured in the test tool configurator. If there is a mismatch, the test tool will not be able to create a C-MOVE request, 
since it verifies if those AE titles are the same.
