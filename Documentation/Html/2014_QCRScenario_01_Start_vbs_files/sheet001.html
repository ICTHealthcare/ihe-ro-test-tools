Test Scenario QCR Start
Instructions for use test scenarios
Rule Overview
Description:
This test scenario tests the create UPS and subscribe to this UPS by the QCR under test. It is the first step in the overall test scenario.
The scenario will test either with a dose or a difference check
Test scenario entry point: None, this is the first QCR test scenario that has to be executed.
Stage: Prepare
   1.  (optional)  Add the RT-dataset you want to use by means of the test tool configuration editor (see Instructions for use test scenarios)
   2.   Note: if the RT Plan for difference or dose check is missing from the test data, the test will abort with an error.
   3.   Start the scenario. Select if it is a dose or difference check.
   4.  The test tool starts listening for incoming messages.
   5.   It will automatically detect the N-Create request and will save the request in the cache directory.
   6.   Then the test tool will send back an acknowledge N-Create-Respons. 
   7.   The test tool automatically detect the N-Action request and will send back an acknowledge N-Action-Respons.
   8.   When the communication is finished, press ‘stop listening and start validation’.
   9.    The test tool validates the N-Create request and the N-Action request.
Expected transactions from actor:
Validation of transaction:
RO-Q1: Create UPS for Quality Check
RO-Q2: Subscribe to UPS Progress Update
