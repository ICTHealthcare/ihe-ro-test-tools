Test Scenario QCP Output Objects Retrieval
Instructions for use test scenarios
Rule Overview
Description:
This scenario tests retrieval of a structured report from the Quality check Performer. Either repeat or retry the retrieval for the succes scenarios 
The QCP configuration is configured with the testtool configuration editor
Test scenario entry point: First execute the QCP Dose check succes or Difference check succes – test scenario.
Stage: Retrieve
   1.   Make sure that the QCP under test has the Structured report available. 
   2.   Start the test scenario, a pop-up window will appear, choose if it a check report with or without issues is expected.
   3.   The test tool sends the C-Move request. (based on the previous received Output Information Sequence of the N-Get respons of RO-Q6)
   4.   The test tool starts listening  for the C-Move response(s) and starts  listening for incoming C-STORE request(s).
   5.   The QCP is expected  to send C-Move response(s) and the C-Store request.
   6.   After the structured report is received ,the test tool will send a N-Action unsubscribe request to the QCP
   7.   The QCP is expected to send a N-Action response confirming that the unsubscribe is acknowledged.
   8.   When the communication is finished, press ‘stop listening and start validation’.
   9.   The test tool will validate all transactions and the Structered Report for DICOM & IHE-compliance.
    (The validation overview can be found in the Rule Overview tab)

Additional technical information
The C-MOVE request that the test tool sends to the QCP consists of references that were found in the N-Get respons of RO-Q6. 
The “Retrieve AE Title” (0008,0054) should be  the same as configured in the test tool configurator.
 If there is a mismatch, the test tool will not be able to create a C-MOVE request.
