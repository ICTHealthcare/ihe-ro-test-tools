Test Scenario QCP Start
Instructions for use test scenarios
Rule Overview
Description:
This test scenario tests the create UPS and subscribe to this UPS by the QCP under test. It is the first step in the overall test scenario.
The scenario will test either with a dose or a difference check
Test scenario entry point: None, this is the first QCP test scenario that has to be executed.
Stage: Prepare
   1.  (optional)  Add the RT-dataset you want to use by means of the test tool configuration editor (see Instructions for use test scenarios)
   2.   Note: if the RT Plan for difference or dose check is missing from the test data, the test will abort with an error.
   3.   Start the scenario. Select if it is a dose or difference check.
   4.   The test tool sends a N-Create-Req to the QCP, in this message there will be a reference to a RT plan of an difference or dose check.
   5.   The test tool automatically detects the response.
   6.   When the UPS is created successfully a N-Action-Req will be send to subscribe to the UPS. 
   7.    The test tool automatically detects the response and starts validating the N-Create and N-Action responses.
Expected transactions from actor:
Validation of transaction:
RO-Q1: Create UPS for Quality Check
RO-Q2: Subscribe to UPS Progress Update
Additional technical information:
The test tool is not an actual Quality Check Requester. The RT Object that are placed in the UPSdata/Input directory
 (by the test tool configuration editor) will be used  for  creating the N-Create-Req. 
Used scenario dataset:
This scenario uses a predefined request which can be modified by the user
