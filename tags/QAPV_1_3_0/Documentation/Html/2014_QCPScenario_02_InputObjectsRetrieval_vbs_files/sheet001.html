Test Scenario QCP Input Objects Retrieval
Instructions for use test scenarios
Rule Overview
Description:
This test scenario allows a QCP actor to retrieve the input objects instances (RT Plan) form the QCR actor.
The QCR emulator configuration and available instances are configured with the testtool configuration editor
Test scenario entry point: First execute the QCP Start – test scenario.
Stage: Prepare
   1.   Make sure that one RT-Plan is present for difference and dose check (see step 1 of the QCP Start test scenario where to place the files)
   2.   Start the test scenario and choose between an dose or difference check.
   3.   The test tool waits for an incoming C-Move request
   4.   Send a C-Move request with the QCP
   5.   The test tool sends C-Move pending responses to the QCP actor and the test tool sends C-Store requests to the QCP actor (based upon the C-MOVE-RQ)
   6.   The QCP Actor should send one or more C-Store responses to the test tool, depending on the C-Move request.
   7.   When the DICOM objects are successfully stored, the test tool will send a C-MOVE success response.
   8.   When the communication is finished, press ‘stop listening and start validation’.
   9.   The test tool validates the C-Move requests and C-Store responses for IHE-compliance. At least one RT-Plan should be received.
    (The validation overview can be found in the Rule Overview tab)
