Test Scenario QCR Check Cancel
Instructions for use test scenarios
Rule Overview
Description:
This scenario tests a cancelled UPS update, retrieval of cancelled output objects instances from the QCP actor.
The QCR emulator configuration and available instances are configured with the testtool configuration editor
Test scenario entry point: First execute the QCR Input Objects Retrieval – test scenario.
Stage: Prepare
   1.   Start the test scenario.
   2.   The test tool sends the N-Event-Report request with the procedure state IN PROGRESS (RO-Q5)
   3.   The QCR will respond with a N-Event-Report response
   4.   The test tool sends the N-Event-Report request with the procedure state CANCELED (RO-Q5)
   5.   The QCR will respond with a N-Event-Report response
   6.   After a cancelled progress the QCR will send a N-Get request to retreive cancelled information why the check was cancelled. 
   7.   The test tool now sends the N-Get respons with an empty output sequence and information about the cancellation
   8.   The QCR sends a N-Action UPS unsubscribe request.
   9.   The test tool sends a N-Action respons that the unsubscribe is acknowledged.
   10.   When the communication is finished, press ‘stop listening and start validation’.
   11.   The test tool validates all transactions for IHE-compliance.
    (The validation overview can be found in the Rule Overview tab)
Expected transactions from actor:
Validation of transaction:
RO-Q5: Update on UPS Progress
RO-Q6: Output Information Sequence Retrieval
RO-Q8: Unsubscribe to UPS Progress
