Test Scenario QCP Check Cancel
Instructions for use test scenarios
Rule Overview
Description:
This test scenario tests the Quality Check Performer when cancelling  the check request.
This scenario an UPS update with the CANCELLED state is expected and optionally put objects instances from the QCP actor.
The QCR emulator configuration and available instances are configured with the testtool configuration editor
Test scenario entry point: First execute the QCP Input Objects Retrieval – test scenario.
Stage: Prepare
   1.   Start the test scenario.
   2.   The test tool starts to listen for incoming messages.
   3.   The QCP is expected to send a N-Event-Report request with the procedure state IN PROGRESS (RO-Q5)
   4.   The test tool will automatically respond with a N-Event-Report response
   5.   The QCP  is expected to send a N-Event-Report request with the procedure state CANCELED (RO-Q5)
   6.   The test tool will automatically respond with a N-Event-Report response
   7.   After a cancelled progress the test tool will send a N-Get request to retreive cancelled information why the check was cancelled. 
   8.   The QCP  is expected to send a N-Get respons with the reason of cancellation and an empty output sequence.
   9.   After that the N-Get respons is received the test tool will send a N-Action unsubscribe request to the QCP
   10.   The QCP sends a N-Action respons that the unsubscribe is acknowledged.
   11.   When the communication is finished, press ‘stop listening and start validation’.
   12.   The test tool validates all transactions for DICOM & IHE-compliance.
    (The validation overview can be found in the Rule Overview tab)
Expected transactions from actor:
Validation of transaction:
RO-Q5: Update on UPS Progress
RO-Q6: Output Information Sequence Retrieval
RO-Q8: Unsubscribe to UPS Progress
Additional technical information
The retrieval of optional  output object(s ) is out of the scope of the IHE profile and is not considered
