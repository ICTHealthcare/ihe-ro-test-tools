Test Scenario QCR Dose Check Success
Instructions for use test scenarios
Rule Overview
Description:
This scenario tests UPS updates, retrieval of output objects instances and retrieval of a structured report from the QCP actor.
The QCR emulator configuration and available instances are configured with the testtool configuration editor
Test scenario entry point: First execute the QCR Input Objects Retrieval – test scenario.
Stage: Prepare
   1.   Make sure that at least one Structured report is present for good check and a check with issues (Use test tool configuration editor to place the files)
   2.   Start the test scenario, a pop-up window will appear, choose if it is a good check or a check with issues.
   4.   The test tool sends the N-Event-Report request with the procedure state IN PROGRESS (RO-Q5)
   5.   The QCR will respond with a N-Event-Report response
   6.   The test tool sends the N-Event-Report request with the procedure state COMPLETED (RO-Q5)
   7.   The QCR will respond with a N-Event-Report response
   8.   After a completed progress the QCR will send a N-Get request to retreive to output information sequence with information where the structured report is placed. 
   9.   The test tool now sends the N-Get respons with information where the structured report is placed.
   10.   The QCR sends the C-Move request. (based on the previous received Output Information Sequence of the N-Get respons of RO-Q6)
   11.   The QCR detects the C-Move response(s) and will wait for incoming C-STORE request(s).
   12.   The test tool sends the C-Store request with as dataset the structured report and C-Move responses
   13.   After that the structured report is sent the test tool will listen for an N-Action unsubscribe request
   14.   The QCR sends a N-Action UPS unsubscribe request.
   15.   The test tool sends a N-Action respons that the unsubscribe is acknowledged.
   16.   When the communication is finished, press ‘stop listening and start validation’.
   17.   The test tool validates all transactions for IHE-compliance.
    (The validation overview can be found in the Rule Overview tab)
Notes:
        C-Move and C-Store dimse services are executed on two different associations.
        This test scenario does not cover validation of an unsafe delivery procedure.(SOP-instances in the Output Information Sequence received when the 
        Procedure Step State was COMPLETED)
        Assumption: QCP sends C-Move Responses for progress update during C-Store procedure. (some implementations of a PACS do not send 
        C-Move Responses until completion)
Expected transactions from actor:
Validation of transaction:
RO-Q5: Update on UPS Progress
RO-Q6: Output Information Sequence Retrieval
RO-Q7: Quality Check Report Retrieval
RO-Q8: Unsubscribe to UPS Progress
Used scenario dataset:
SOP Class support to QCR:
   Basic Text Structured Report
Additional technical information
The C-MOVE request that the test tool sends to the QCP consists of work items that were found in the N-Get respons of RO-Q6. In the output 
information sequence should be a reference to the structured report Storage file that is stored on the QCP. The “Retrieve AE Title” (0008,0054) should have 
a value that is the same as configured in the test tool configurator. If there is a mismatch, the test tool will not be able to create a C-MOVE request, 
since it verifies if those AE titles are the same.
